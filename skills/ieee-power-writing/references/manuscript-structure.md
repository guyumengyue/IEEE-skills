# IEEE PES Manuscript Structure

## Abstract

Pattern:

`problem -> gap -> method -> main result -> implication -> boundary`

Keep it one paragraph and about 150-200 words. Avoid citations, footnotes,
displayed equations, tables, and unexplained abbreviations.

## Introduction

1. Define the engineering problem and operating context.
2. Explain why existing methods, models, data, or controls are insufficient.
3. State the unresolved technical gap.
4. State contributions as testable claims.
5. Preview the evidence, not every result detail.

## Method

Use reproducible order:

1. System and assumptions.
2. Notation and variables.
3. Formulation, architecture, or control logic.
4. Solver, estimator, learning method, or hardware implementation.
5. Complexity, convergence, or deployment considerations when relevant.

## Results

Start with the case system and evaluation protocol, then build an evidence
ladder: main comparison, diagnostic analysis, ablation, sensitivity, stress
test, and practical implication.

## Conclusion

State what was demonstrated, what evidence supports it, and where the claim
stops. Do not add new results.
